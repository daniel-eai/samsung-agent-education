현재 프로젝트의 Step 1 분석 결과, AORR.md, MEMORY.md, 기존 저장소 파일을 읽고 Self-Correcting TDD 루프를 실행해줘.

이번 단계의 목표는 GitHub Pages에서 실제로 사용할 수 있는 정적 개인 프로페셔널 웹사이트를 완성하는 것이야.

프로페셔널 웹사이트의 모든 필수 화면과 기능, 반응형 디자인,
Games 탭, 키보드와 모바일 터치로 조작 가능한 지렁이 게임을
완전히 구현하고 테스트한 뒤 GitHub Pages에 최초 배포해야 한다.

웹사이트 또는 게임 기능이 미완성된 상태에서는
DEPLOY_READY, DEPLOYED 또는 PASSED로 종료하지 마.

## 필수 구현 범위

다음 항목을 모두 구현해:

1. 프로젝트 루트의 정적 웹사이트 파일
   - index.html
   - styles.css
   - script.js
   - 필요한 경우 game.js와 assets 디렉토리

2. 개인 프로페셔널 웹사이트
   - Home 또는 Hero 영역
   - About 영역
   - Projects, Experience, Research 중 기존 자료에 맞는 영역
   - Contact 영역
   - 상단 내비게이션
   - Games 탭

3. 반응형 디자인
   - 데스크톱 지원
   - 태블릿 지원
   - 모바일 지원
   - 모바일 내비게이션
   - 가로 스크롤 방지
   - 모바일에서 충분한 버튼 및 터치 영역 제공

4. 지렁이 게임 전체 기능
   - 게임 시작
   - 자동 이동
   - 음식 생성
   - 음식 섭취 및 성장
   - 점수 증가
   - 벽 충돌
   - 자기 몸 충돌
   - 게임 오버
   - 일시정지 및 재개
   - 다시 시작
   - 최고 점수
   - 중복 게임 루프 및 타이머 방지
   - 반대 방향으로 즉시 전환되는 입력 방지

5. 게임 조작
   - 키보드 방향키
   - W, A, S, D
   - 모바일 화면 방향 버튼
   - 가능한 경우 모바일 Swipe
   - 게임 조작 중 의도하지 않은 페이지 스크롤 방지

6. Step 1의 다음 영역에 추가 요구사항이 작성되어 있다면 반드시 반영
   - [게임 추가 기능:]

기존 사이트가 있다면 전체를 무조건 덮어쓰지 말고,
기존 콘텐츠와 사용 가능한 구조를 최대한 보존해.

확인되지 않은 경력, 프로젝트, 연구 실적, 연락처 등의 정보를 임의로 생성하지 말고
[사람 확인 필요]로 표시해.

## 실행 순서

전체 개발 업무를 다음과 같은 작은 루프로 나누어 순차적으로 실행해:

1. MEMORY.md의 목표, 범위, 가드레일 확인
2. 기존 저장소 파일과 Git 상태 확인
3. Git remote와 대상 저장소 확인
4. 변경 전 상태를 Verifier로 확인
5. 정적 사이트 기본 구조 구현
6. 프로페셔널 콘텐츠 영역 구현
7. 데스크톱 레이아웃 구현
8. 태블릿 및 모바일 반응형 구현
9. 내비게이션과 Games 탭 구현
10. 지렁이 게임 상태 및 렌더링 구현
11. 음식, 성장, 점수 구현
12. 충돌, 게임 오버, 재시작 구현
13. 키보드 조작 구현
14. 모바일 터치 조작 구현
15. 접근성 및 사용성 검증
16. 전체 회귀 테스트
17. GitHub Pages 호환성 검증
18. MEMORY.md 실행 기록 업데이트
19. 사용자에게 배포 승인 요청
20. 사용자 승인 후 GitHub Pages 배포
21. 배포된 사이트에서 최종 검증

각 개발 루프에서는 다음 순서를 지켜:

1. 이번 루프의 최소 완료 기준 결정
2. 변경 전 Verifier 실행
3. 필요한 최소 파일만 수정
4. 동일한 Verifier 다시 실행
5. 결과와 실패 로그 관찰
6. 실패 원인 분류
7. 필요한 경우 최소 수정 후 Retry
8. MEMORY.md에 실행 결과 기록
9. 통과하면 다음 개발 루프로 이동

실패 원인은 다음 중 하나로 분류해:

- CSS_RESPONSIVE
- JAVASCRIPT
- GAME_STATE
- GAME_LOGIC
- GAME_RENDERING
- GAME_CONTROL
- ACCESSIBILITY
- CONTENT
- TEST
- ENVIRONMENT
- GITHUB_PERMISSION
- DEPLOYMENT
- UNKNOWN

## Retry 정책

- 하나의 오류에 대해 최대 3회 Retry
- 동일한 오류 fingerprint가 2회 반복되면 중지
- 한 번의 Retry에서는 하나의 실패 원인만 수정
- 관련된 최소 파일만 변경
- 테스트 삭제 또는 완화 금지
- 기능을 삭제해서 테스트를 통과시키는 행위 금지
- 불필요한 전체 코드 재작성 금지
- 권한이나 환경 문제를 코드 변경으로 우회하지 않음

Retry 한계에 도달하거나 요구사항이 불명확하면
상태를 HITL_REQUIRED로 설정하고 중지해.

## Verifier

프로젝트 환경에서 실제 사용할 수 있는 검증 도구를 먼저 확인해.
존재하지 않는 테스트 명령어를 임의로 만들지 마.

최종적으로 다음 항목을 모두 검증해:

### 파일 및 정적 사이트

- 프로젝트 루트에 index.html 존재
- styles.css 존재
- script.js 존재
- 필요한 경우 game.js 존재
- HTML, CSS, JavaScript 정상 연결
- 잘못된 로컬 절대 경로 없음
- GitHub Pages에서 사용할 수 있는 상대 경로 사용
- github_token.txt가 웹사이트 코드와 연결되지 않음

### 웹사이트

- 주요 프로페셔널 영역 구현 완료
- 내비게이션 정상 동작
- Games 탭 정상 동작
- 깨진 내부 링크 없음
- 브라우저 콘솔에 치명적인 오류 없음
- 기존 콘텐츠가 불필요하게 삭제되지 않음

### 반응형

가능하면 다음 viewport에서 확인해:

- 모바일: 약 375px
- 태블릿: 약 768px
- 데스크톱: 약 1440px

다음을 검증해:

- 가로 스크롤 없음
- 텍스트 및 UI 잘림 없음
- 모바일 메뉴 정상
- 게임 보드가 화면을 벗어나지 않음
- 모바일 조작 버튼의 터치 영역이 충분함

### 지렁이 게임

- 게임 시작
- 자동 이동
- 음식 생성
- 음식이 지렁이 몸 위에 생성되지 않음
- 음식 섭취
- 지렁이 성장
- 점수 증가
- 최고 점수
- 벽 충돌
- 자기 몸 충돌
- 게임 오버
- 일시정지 및 재개
- 다시 시작
- 방향키 조작
- WASD 조작
- 모바일 방향 버튼 조작
- 구현한 경우 Swipe 조작
- 반대 방향 즉시 전환 방지
- 중복 게임 타이머 방지
- Games 탭을 반복해서 열어도 게임 루프가 중복되지 않음
- Step 1의 추가 게임 기능 정상 동작

### 로컬 서버

가능하면 다음 명령어 또는 현재 환경에서 사용 가능한 정적 서버를 사용해:

python3 -m http.server

다음을 확인해:

- index.html HTTP 200
- CSS HTTP 200
- JavaScript HTTP 200
- 정적 assets HTTP 200
- 브라우저에서 페이지 정상 로드

브라우저 자동화 또는 화면 검증을 수행할 수 없는 항목은
[사람 확인 필요]로 표시해.

가능하면 Claude Code CLI를 독립 Verifier로 사용해.

Claude Code CLI에서 Sonnet 5가 실제 사용 가능한지 먼저 확인해.
사용 가능하면 Sonnet 5를 사용해.

사용할 수 없다면 현재 사용 가능한 Sonnet 모델을 사용하고,

Claude Code CLI를 사용할 수 없다면 해당 이유를 MEMORY.md에 기록하고,
사용 가능한 다른 Verifier를 이용해 계속 진행해.

## 사용자 배포 승인

모든 구현과 로컬 검증이 이상 없이 완료되면,
상태를 DEPLOY_APPROVAL_REQUIRED로 설정해.

이 시점에는 아직 commit, push 또는 배포하지 마.

Git remote를 확인해 실제 GitHub 사용자명과 저장소 이름을 구한 뒤,
사용자에게 다음과 같이 질문하고 중지해:

“프로페셔널 웹사이트와 지렁이 게임의 전체 구현 및 로컬 검증이 완료되었습니다.

대상 저장소:
https://github.com/[GitHub 사용자명]/[GitHub 사용자명].github.io

예상 배포 주소:
https://[GitHub 사용자명].github.io

현재 버전을 GitHub Pages에 최초 배포할까요?”

현재 Step 1의 저장소가
https://github.com/daniel-eai/daniel-eai.github.io.git 인 경우에는
다음 주소를 사용해:

- 대상 저장소: https://github.com/daniel-eai/daniel-eai.github.io
- 배포 주소: https://daniel-eai.github.io

사용자가 명시적으로 승인하기 전에는 다음 작업을 하지 마:

- commit
- push
- GitHub Pages 배포
- GitHub 저장소 설정 변경

## 승인 후 GitHub Pages 배포

사용자가 명시적으로 배포를 승인한 경우에만 다음을 수행해:

1. Git remote에서 사용자명과 저장소를 다시 확인
2. 저장소가 [GitHub 사용자명]/[GitHub 사용자명].github.io 형식인지 확인
3. git status와 Git diff 확인
4. github_token.txt가 Git 추적 또는 stage 대상이 아닌지 확인
5. env_settings.txt, .env 및 비밀정보 파일이 포함되지 않았는지 확인
6. 필요한 경우 비밀정보 파일을 .gitignore에 추가
7. 코드와 Git diff에 토큰 문자열이 포함되지 않았는지 검사
8. 배포에 필요한 파일만 stage
9. 적절한 commit 메시지 작성
10. commit 생성
11. 현재 디렉토리의 github_token.txt에 있는 token="" 값을 GitHub 인증에 사용
12. GitHub Pages 대상 브랜치에 push
13. GitHub Pages 배포 완료 확인
14. https://[GitHub 사용자명].github.io 에서 HTTP 200 확인
15. 배포된 웹사이트와 지렁이 게임 전체 기능 재검증
16. 배포 결과를 MEMORY.md에 기록
17. 최종 상태를 DEPLOYED로 설정

토큰 사용 시 다음 규칙을 반드시 지켜:

- 토큰 값을 출력하지 마
- 토큰을 사용자에게 보여주지 마
- 토큰을 HTML, CSS 또는 JavaScript에 저장하지 마
- 토큰을 MEMORY.md에 기록하지 마
- 토큰을 Git commit에 포함하지 마
- 토큰을 영구 Git remote URL에 포함하지 마
- 토큰을 로그나 명령 출력에 노출하지 마
- 임시 인증 파일 또는 환경 변수를 만들었다면 배포 후 즉시 제거해

저장소 형식이 다르거나,
GitHub Pages 설정 변경이 필요하거나,
인증 또는 권한 문제가 발생하면 임의로 우회하지 마.

상태를 HITL_REQUIRED 또는 BLOCKED로 설정하고
[사람 확인 필요]로 보고해.
